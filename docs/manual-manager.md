# 教材制作進捗管理 — 簡易マニュアル（マネージャー向け）

---

## このアプリの目的

教材（動画レッスン）の制作進捗を **リアルタイムで可視化** するWEBアプリです。
1レッスンにつき **23工程**（前工程13 + 後工程10）をチェックボックスで管理し、遅延の早期発見・ボトルネック把握を可能にします。

**アクセスURL:** https://daisuketaguchi-coder.github.io/progress-tracker/

---

## 3つの画面

| タブ名 | 対象ユーザー | 概要 |
|--------|------------|------|
| **進捗管理** | 全員 | メイン画面。レッスンごとのカード一覧と工程マトリクス |
| **新規レッスン登録** | 担当者 | レッスンの新規登録フォーム（Google Drive自動フォルダ作成付き） |
| **定例レビュー** | マネージャー | KPI・ボトルネック・遅延アラートを一覧するダッシュボード |

---

## 基本操作フロー

```
① 新規レッスン登録  →  ② 日々の進捗チェック  →  ③ レビュー依頼  →  ④ 定例レビューで全体把握
```

### ① 新規レッスン登録

1. 「新規レッスン登録」タブを開く
2. **担当者名**と**レッスン名**を入力（必須）
3. 必要に応じて開始日・納期・リリース日を設定
4. 「登録する」をクリック
5. Google Driveにテンプレートフォルダが自動コピーされる

### ② 日々の進捗チェック

1. 「進捗管理」タブを開く
2. レッスンカードの **前工程** または **後工程** セクションを展開
3. 完了した工程の **チェックボックスをON** にする
4. 進捗率が自動計算される（プログレスバーに反映）

### ③ レビュー依頼（Slack通知）

1. マネージャーCK工程の横にある「**レビュー依頼**」ボタンをクリック
2. Slackに通知が自動送信される
3. 対象工程:
   - アウトラインマネージャーCK
   - スライド構成案マネージャーCK
   - 台本マネージャーCK

### ④ 定例レビューで全体把握

1. 「定例レビュー」タブを開く
2. ダッシュボードで全体状況を確認（詳細は後述）

## マネージャー向け：定例レビュー画面の読み方

### 総論レポート（画面上部）

| セクション | 見るべきポイント |
|-----------|----------------|
| **ナラティブサマリー** | 全体状況・ボトルネック・遅延・高負荷担当者が箇条書きで自動生成 |
| **KPIカード（5枚）** | 全教材数 / リリース済み / リリース間近 / 進行中 / 未着手 |
| **進捗ゲージ（3つ）** | 全体・前工程・後工程の平均進捗率を視覚表示 |
| **ボトルネック分析** | 最も多くのレッスンが滞留している工程TOP3 |

### アラートバー

| 色 | 意味 |
|----|------|
| **赤** | 納期超過のレッスンあり（即対応が必要） |
| **黄** | 納期まで7日以内かつ進捗80%未満（要注意） |
| **緑** | 遅延・警告なし |

### その他のセクション

- **教材ステータス一覧** — リリース済み / リリース間近 / 進行中 / 未着手 の4列で一覧表示
- **担当者別稼働状況** — 担当者ごとの件数・平均進捗・負荷レベル（軽/中/高）
- **遅延アラート** — 遅延・警告レッスンの詳細（停滞箇所・超過日数）

## よく使う機能

| 操作 | 方法 |
|------|------|
| **担当者でフィルタ** | 画面右上のプルダウンで担当者を選択 |
| **担当者名・レッスン名の編集** | カード上の名前にホバー → ✏️ → 入力 → Enter |
| **工程マトリクスの表示切替** | 「前工程」「後工程」ボタンで切替。ソート順も変更可能 |
| **レッスンの削除** | カード右上の × ボタン（確認ダイアログあり） |

## 進捗率の計算ロジック

| 指標 | 計算式 |
|------|--------|
| 全体進捗率 | チェック済み工程数 / 23 × 100% |
| 前工程進捗率 | 前工程チェック済み / 13 × 100% |
| 後工程進捗率 | 後工程チェック済み / 10 × 100% |
